# STATECH Website — Deployment Guide

Your site is a **Node.js application** (not static HTML). It must run with `npm start` so the API, admin login, and contact form work.

---

## Option 1: Run locally (fastest — ready now)

### Windows — double-click
```
start.bat
```

### Or from terminal
```bash
npm install
npm start
```

| URL | Purpose |
|-----|---------|
| http://localhost:3000 | Public website |
| http://localhost:3000/login | Team admin login |
| http://localhost:3000/admin | Admin dashboard (after login) |

**Default login:** `admin@statech.com` / `changeme123` — change this immediately in Settings.

To access from other devices on your network, use `http://<your-computer-ip>:3000`.

---

## Option 2: Docker (production-ready container)

Requires [Docker Desktop](https://www.docker.com/products/docker-desktop/).

```bash
docker compose up -d --build
```

Site runs at http://localhost:3000. Data persists in a Docker volume.

---

## Option 3: Render.com (free public hosting)

1. Push this project to **GitHub** (see below).
2. Sign up at [render.com](https://render.com).
3. **New → Blueprint** → connect your repo (uses `render.yaml` automatically).
4. Set environment variables when prompted:
   - `ADMIN_EMAIL` — your team email
   - `ADMIN_PASSWORD` — a strong password
5. Deploy. Render gives you a URL like `https://statech-website.onrender.com`.

`SESSION_SECRET` is auto-generated by Render.

---

## Option 4: Railway / Fly.io / VPS

Any Node.js host works:

- **Build:** `npm install`
- **Start:** `npm start`
- **Port:** set `PORT` env var (host usually provides this)
- **Health check:** `GET /api/health`

Copy `.env.example` to `.env` on the server and set all values.

---

## Environment variables

| Variable | Required | Description |
|----------|----------|-------------|
| `PORT` | No | Server port (default 3000) |
| `HOST` | No | Bind address (default 0.0.0.0) |
| `NODE_ENV` | Yes (prod) | Set to `production` when live |
| `SESSION_SECRET` | Yes | Long random string for session security |
| `ADMIN_EMAIL` | Yes | First admin account email |
| `ADMIN_PASSWORD` | Yes | First admin account password |

## Before going live — checklist

- [ ] Change `ADMIN_PASSWORD` in `.env` or hosting dashboard
- [ ] Set a unique `SESSION_SECRET` (already generated in your `.env`)
- [ ] Update contact info in Admin → Settings
- [ ] Add your real team members in Admin → Team Access
- [ ] Replace portfolio images in `assets/images/portfolio/` with real screenshots

---

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Blank page / translations missing | Don't open HTML files directly — use `npm start` |
| Admin login fails | Check `data/users.json` exists; delete it to recreate admin from `.env` |
| Contact form doesn't save | Server must be running (`npm start`) |
| Session lost on Render | `NODE_ENV=production` and HTTPS are required (Render handles HTTPS) |
